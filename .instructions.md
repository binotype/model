# Repository Guidelines Configuration

## Core Guidelines Integration

This repository contains comprehensive coding guidelines in the `guidelines/` folder. These guidelines MUST be considered for all development work.

### Automatic Guideline Loading

The following guidelines are available and should be loaded based on context:

1. **TypeScript Guidelines** ([guidelines/typescript.md](guidelines/typescript.md))
   - Applies to: `**/*.ts` (excluding test files)
   - Triggers: typescript, interface, namespace, class, factory
   - Core pattern: Interface + Namespace for domain objects

2. **Testing Guidelines** ([guidelines/testing.md](guidelines/testing.md))
   - Applies to: `**/*.spec.ts`
   - Triggers: test, spec, jest, describe, it
   - Primary approach: Snapshot testing with parameterized tests

3. **Module Export Guidelines** ([guidelines/module-exports.md](guidelines/module-exports.md))
   - Applies to: `**/index.ts`
   - Triggers: export, import, index
   - Pattern: Alias imports to avoid naming conflicts

### Guidelines Reference Index

For quick navigation, always refer to [guidelines/README.md](guidelines/README.md) which provides:
- File type mappings

- Quick reference patterns

### Mandatory Guidelines Consideration

When working in this repository:

1. **ALWAYS** check the guidelines folder first before implementing
2. **LOAD** the appropriate guideline based on file type and task
3. **FOLLOW** the established patterns and conventions
4. **REFERENCE** the quick navigation in guidelines/README.md

### Critical Patterns to Enforce

- **Single Return Rule**: Always use single return statement with `result` variable
- **Interface + Namespace**: Primary pattern for domain objects
- **Factory Functions**: Always named `load()` in namespaces
- **Import Aliases**: Use `import { X as _X }` pattern to avoid conflicts
- **Test Organization**: Snapshot testing with `it.each()` for parameterized tests

## Context Loading Strategy

The guidelines are designed with optimal context loading:
- **High priority** guidelines load automatically for relevant file types
- **Medium context size** to balance completeness with performance
- **Trigger words** activate specific guidelines when mentioned
- **File pattern matching** ensures relevant guidelines are applied

This configuration ensures that any AI agent working in this repository will automatically consider and apply the established coding standards and patterns.
